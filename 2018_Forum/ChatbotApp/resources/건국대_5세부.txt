[지능정보 서비스를 위한 클라우드 환경에서의 고성능 GPU 가상화 연구]

"클라우드 기반 GPU 가상화 인프라"

∙ 하이퍼바이저기반 GPU 가상화 솔루션인 인텔의 GVT-g는 다수의 가상머신에게 모든 GPU 기능을 제공함
∙ 가상머신간의 글로벌 그래픽스 메모리 공유를 통해 최대 15개의 가상머신을 지원할 수 있음
∙ 글로벌 그래픽스 메모리 공유로 인해 GPU 문맥 전환때 할당 받은 메모리 슬롯에 대한 페이지 테이블 복사가 발생하여 성능저하의 원인이 됨

"클라우드 환경에서 GPU 문맥전환 오버헤드 최소화 기법"

▶ GPU 메모리 모니터
∙ 각 가상머신마다 할당된 가상 GPU(vGPU)로부터 GPU 사용 정보들을 주기적으로 수집하여 GPU 메모리 할당자에게 전달함

▶ GPU 메모리 할당자
∙ vGPU의 GPU 사용 패턴에 따라 두 가지 알고리즘을 바탕으로 GPU 문맥 전환 오버헤드를 최소화하도록 vGPU를 GPU 메모리에 배치함
∙ 가상머신 생성 또는 종료, 그리고 주기적으로 GPU를 사용하지 않는 vGPU를 분류하여 GPU 메모리 공유를 최소화하도록 vGPU를 재배치함
∙ 시시각각 변화하는 vGPU의 GPU 사용률을 고려하여 vGPU를 재배치함

"성능 평가"

∙ 가상머신간의 GPU 사용률의 편차가 크지 않은 경우, 가상머신이 할당받은 GPU 메모리 크기 기반으로 vGPU
를 메모리에 매핑했을 때 성능이 최대 39% 향상함

∙ 가상머신간의 GPU 사용률의 편차가 큰 경우, vGPU를 GPU 메모리 크기 기반으로 배치했을 때 최대 21%, GPU
사용률을 고려하여 vGPU를 배치했을 때 최대 41%의 성능 향상을 보임
